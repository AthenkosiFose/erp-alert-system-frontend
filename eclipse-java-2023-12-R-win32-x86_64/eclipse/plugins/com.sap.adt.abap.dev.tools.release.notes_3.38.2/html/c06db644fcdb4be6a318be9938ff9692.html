ABAP Development Tools for Eclipse

These release notes are a general overview of the most significant changes relating to ABAP development tools for Eclipse features in the context of ABAP development and the ABAP programming models.

 Displaying Memory Consumption of a Debugged ABAP Program

When debugging, you can now display the memory consumption of a debugged ABAP program in the new ABAP Memory (Debugger) view.

Sample of an ABAP Debugger Preview Where the ABAP Memory View Is Opened

The memory consumption is displayed in accordance to the respective memory area and the current debugger position. This enables you to investigate changes in the memory consumption of your ABAP application while debugging it.

In addition, you can ...

  • display the changes of the memory consumption between a baseline and the current debugger position.

  • update the displayed memory consumption automatically or manually while debugging.

  • create a memory snapshot for deeper investigations of the memory consumption.

  • run the garbage collector.

For more information, see

  • ABAP Memory (Debugger) View

  • Creating a Memory Snapshot

  • Displaying the Current Memory Consumption

Displaying Component Groups in the Variables View While Debugging

If you have a structure with one or more component groups, you can now display the component group(s) in the Variables view.

Example

The component group city is added to the view and displayed hierarchically in the structure address.

Display of a Component Group in the Variables View

For more information, see Displaying Component Groups

Working with the New Where-Used Dialog

You can now use the new where-used pre-execution dialog to pre-filter your where-used search results. Restricting the search scope helps you to concentrate on usages that are relevant for your current use case.

For more information, see Where-Used Dialog.

Rework of the "Ignore White Space" Option in the Compare Editor

In the past, the Ignore White Space option in the compare editor ignored all white spaces, including the significant ones that can influence the program logic, such as string literals.

Now, the significant white spaces are visible.

Sample for displaying white spaces in string literals despite using the "Ignore White Space" option
Note Currently, this feature is only supported for

  • ABAP (inclusive ABAP Managed Database Procedures (AMDP), Embedded Native SQL (EXEC SQL))

  • ABAP CDS-related languages, such as DDLX, DDLS, and so on

For more information, see also Improved "Ignore Whitespace" in Java Compare

Comparing with Previous Revision

You can now compare previous revision in the History view using a new menu entry by selecting any revision.

Comparing Multiple Objects - Displaying Messages in a Single Pop-up

You can now view all the messages in a single pop-up message while comparing large sets of ABAP repository objects across two ABAP projects from the ADT project explorer. If the objects are not identical, the changes are displayed in the compare editor.
